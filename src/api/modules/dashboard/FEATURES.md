# FEATURES - `dashboard`

## Estado atual

O modulo hoje atua como fatia de garantia para metricas comerciais, usando dados de sales e recovery. A visao executiva ainda parece parcial, mas tem potencial alto para tomada de decisao diaria.

## Features de alto valor

| Prioridade | Feature | Status | Valor para usuario | Criterio de sucesso |
|------------|---------|--------|--------------------|---------------------|
| P0 | Receita nova vs receita recuperada | parcial | Dono entende crescimento real sem misturar recuperacao com venda nova. | Dashboard separa valores por origem e status pago. |
| P0 | Funil operacional unificado | recomendado | Mostra onde conversas, carrinhos, propostas e pagamentos travam. | Gestor identifica gargalo em ate 1 minuto. |
| P0 | Saude da operacao por modulo | recomendado | Cliente sabe se WhatsApp, IA, agenda, pagamento ou estoque estao falhando. | Queda em tickets de "nao sei o que aconteceu". |
| P1 | Ranking de canais/campanhas | recomendado | Ajuda investir no canal que gera mais retorno. | Receita e conversao por canal ficam visiveis. |
| P2 | Forecast financeiro | nao recomendado agora | Valor futuro, mas depende de dados historicos consistentes. | Priorizar apos metricas base ficarem confiaveis. |

## Observacao

Dashboard nao deve ser vaidade. Deve responder: o que vendeu, o que recuperou, onde travou e o que fazer agora.
