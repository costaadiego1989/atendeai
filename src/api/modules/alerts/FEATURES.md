# FEATURES - `alerts`

## Estado atual

O modulo cria e processa lembretes internos com horario, recorrencia simples e protecoes anti-spam. Ele gera valor quando evita esquecimento de tarefas operacionais ligadas a atendimento, vendas e recuperacao.

## Features de alto valor

| Prioridade | Feature | Status | Valor para usuario | Criterio de sucesso |
|------------|---------|--------|--------------------|---------------------|
| P0 | Lembretes recorrentes confiaveis | parcial | O time nao perde follow-ups importantes. | Lembretes disparados no horario correto e sem duplicidade. |
| P1 | Painel de lembretes vencidos | recomendado | Gestor identifica gargalos e tarefas esquecidas. | Queda no volume de lembretes vencidos por usuario. |
| P1 | Lembretes a partir de conversa | recomendado | Atendente cria follow-up sem sair do contexto do cliente. | Aumento de follow-ups criados a partir do inbox. |
| P2 | Escalonamento automatico para gestor | recomendado | Tarefas criticas nao ficam invisiveis. | Tarefas vencidas acima de SLA geram notificacao ao responsavel. |
| P2 | Regras complexas de calendario | nao recomendado agora | Pode ajudar casos especificos, mas nao e necessidade central. | Reavaliar se houver demanda frequente por agendas complexas. |

## Observacao

O modulo deve continuar simples: lembrar a pessoa certa, no canal certo, sem spam.
